# HIP-295: Hanzo AI Impact Thesis

## Executive Summary

Hanzo AI (Techstars '17) exists to **accelerate high-growth ventures with AI**—partnering with early-stage founders for 10+ years to drive transformative outcomes. We are the creation and acceleration layer of a coordinated ecosystem—alongside Lux (finance) and Zoo (mission)—designed to prove that AI-powered ventures can create net-positive outcomes for humanity and the planet.

Our thesis: **Accelerate projects with AI. Partner for the long term. Technology should augment, not extract.**

## The Problem We're Solving

### Founders Need More Than Capital

| Problem | Consequence |
|---------|-------------|
| **Short-term VC** | Pressure for quick exits, not lasting companies |
| **AI expertise gap** | Founders can't access frontier AI capabilities |
| **Siloed networks** | Limited access to blockchain, impact, and enterprise connections |
| **Build vs. buy confusion** | Wasted resources on solved problems |
| **Safety as afterthought** | Harms emerge post-deployment |

### The Opportunity

The best founders need long-term partners who bring:
- **AI acceleration** (reduce time-to-market with frontier AI)
- **Network access** (Hanzo AI, Lux Blockchain, Zoo Impact ecosystems)
- **10+ year commitment** (partner through pivots, scale, and beyond)
- **Responsible technology** (safety by design, not afterthought)

### LuxDAO Governance Stack

The ecosystem governance is built on the **LuxDAO contracts** at [`github.com/luxfi/dao`](https://github.com/luxfi/dao):

#### Core Contracts (`contracts/`)

| Contract | Path | Purpose |
|----------|------|---------|
| **SystemDeployerV1** | `singletons/SystemDeployerV1.sol` | CREATE2-based orchestrator for complete DAO system deployment |
| **ModuleAzoriusV1** | `deployables/modules/ModuleAzoriusV1.sol` | Azorius Protocol governance module for Gnosis Safe integration |
| **ModuleFractalV1** | `deployables/modules/ModuleFractalV1.sol` | Parent-child DAO relationships for sub-DAOs |
| **StrategyV1** | `deployables/strategies/StrategyV1.sol` | Core voting strategy with EIP-7201 namespaced storage |
| **VotesERC20V1** | `deployables/erc20/VotesERC20V1.sol` | Governance token with ERC20Votes, transfer locking |
| **FreezeVoting*** | `deployables/freeze-voting/` | Parent DAO freeze capability (Multisig, Azorius, Standalone) |

#### Key Architecture Patterns

- **Zodiac Integration**: All modules extend Gnosis Safe via `GuardableModule`
- **EIP-7201 Storage**: Namespaced storage pattern for upgrade safety
- **UUPS Upgradeable**: Universal Upgradeable Proxy Standard
- **CREATE2 Deployment**: Deterministic addresses via `SystemDeployerV1`
- **ERC-4337 Gasless Voting**: Light Account support in `StrategyV1`

#### Proposer Adapters

| Adapter | Purpose |
|---------|---------|
| **ProposerAdapterERC20V1** | Token-gated proposal creation |
| **ProposerAdapterERC721V1** | NFT-gated proposal creation |
| **ProposerAdapterHatsV1** | Role-based proposals via Hats Protocol |

#### Voting Configuration

Each DAO's voting system consists of:
- **VotingWeight** contracts (ERC20/ERC721): Calculate voting power
- **VoteTracker** contracts: Record votes, prevent double-voting
- **Quorum threshold**: Minimum YES + ABSTAIN votes
- **Basis numerator**: Required approval percentage (50-100%)

### Fork & Deploy Your Own

The entire governance stack is **open source** and designed for anyone to fork and deploy:

```bash
# Clone the DAO contracts
git clone https://github.com/luxfi/dao

# Deploy your own governance system
cd dao/contracts
npm install
npx hardhat run scripts/deploy.ts --network <your-network>
```

**Or deploy via SystemDeployerV1** for full DAO setup including:
- Governance tokens with initial allocations
- Proposer adapters for your token/NFT/role requirements
- Voting strategy with your quorum and approval thresholds
- Azorius module connected to your Safe
- Optional: Fractal module for sub-DAO hierarchy
- Optional: Freeze voting for parent-child control
